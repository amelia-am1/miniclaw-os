
# Runtime: Node.js ONLY

DO NOT mix runtimes. This project uses Node.js exclusively.

- NEVER use `bun:sqlite`, `bun:test`, `Bun.serve()`, `Bun.file`, or any `bun:*` imports
- Use `better-sqlite3` for SQLite
- Use `vitest` for tests
- Use `node:fs`, `node:path`, `node:child_process` for system APIs
- Use `bun install` for dependency installation only (it's faster) but ALL runtime code must be Node-compatible
- Use `bun test` as a test runner (it runs vitest) but import from `vitest`, not `bun:test`

## File naming

- Use kebab-case for all files: `setup-wizard.tsx`, `step-email.tsx`
- NEVER use PascalCase filenames like `SetupWizard.tsx`

## No AI attribution

Never add `Co-Authored-By`, `Generated by`, or any AI/Claude credit to commit messages, code comments, or file headers.

## Web app

The board web app runs on Next.js (standalone mode) with `better-sqlite3` for the database.
It serves on port 4220 and includes the setup wizard, settings, and brain board.

## SQLite

Single source of truth for the board schema: `plugins/mc-board/schema.sql`
Seed data: `plugins/mc-board/seed.sql`

## Openclaw plugins

- Each plugin MUST have `openclaw.plugin.json` with an `id` field
- Each plugin MUST have `package.json` with `"openclaw": { "extensions": ["./index.ts"] }`
- Config schema `required` should be empty `[]` — plugins with missing keys get disabled, not rejected
- Plugins are pre-built in `release.sh` (deps installed, native modules compiled) and shipped in the zip
- install.sh copies pre-built plugins directly to `~/.openclaw/extensions/` and writes `openclaw.json` entries — no `openclaw plugins install` CLI needed
